About the Language

What is Axon?

Axon is a new systems programming language purpose-built for machine learning and AI workloads. It bridges the gap between the safety and performance of systems languages and the ergonomics demanded by modern ML workflows.

From data loading through training to deployment — Axon provides a unified, compiled, and verifiable pipeline with zero compromises on performance.

  • Rust-inspired memory safety through ownership and borrowing — zero runtime cost
  • First-class tensor types with compile-time shape verification
  • Native GPU compilation via LLVM/MLIR backend
  • AI-agent-friendly syntax — strict, verbose, unambiguous, with machine-readable errors
  • Full ML pipeline support from data loading through training to deployment

train.axon 
fn train_epoch(
    model: &mut NeuralNet,
    data: DataLoader,
    optimizer: &mut Adam,
) -> Float32 {
    let mut total_loss = 0.0;
    for batch in data {
        let (inputs, targets) = batch;
        let predictions = model.forward(inputs);
        let loss = cross_entropy(predictions, targets);
        total_loss += loss.item();
        loss.backward();
        optimizer.step();
        optimizer.zero_grad();
    }
    return total_loss / data.len() as Float32;
}
Philosophy

Key Design Decisions

Every choice in Axon serves one goal: make ML workloads safe, fast, and verifiable — from compile time to deployment.

🔒

Ownership Model

Rust-style ownership & borrowing. No garbage collector. Memory is freed deterministically when the owner goes out of scope — zero runtime overhead.

📐

Compile-Time Shape Checking

Tensor<Float32, [batch, 784]> catches dimension mismatches before runtime. Dynamic dimensions supported with ?.

⚡

LLVM/MLIR Backend

Same compiler stack as Rust and Mojo. Compiles to native machine code + GPU kernels. C-level performance with zero compromises.

🤖

AI-Agent First

Strict, unambiguous syntax designed for machine consumption. Machine-readable JSON error output. Structured error codes with fix suggestions.

🎯

Smart GPU Dispatch

The compiler automatically decides CPU vs GPU based on operation size. Explicit control available with @gpu / @cpu annotations.

🛡️

No Exceptions, No Null

Result<T, E> for errors, Option<T> for optionality. If it compiles, it works. Algebraic error handling throughout.

Getting Started

CLI Usage

The axonc compiler supports parsing, lexing, and AI-agent-friendly JSON error output.

terminal 
# Parse a source file and print the AST
$ axonc parse hello.axon

# Tokenize a source file and print all tokens
$ axonc lex hello.axon

# JSON error output — designed for AI agents & tooling
$ axonc parse --error-format=json hello.axon

# Run all compiler tests
$ cargo test
running 102 tests ... ok
