Major Findings (3)

⚠️ M-001: Let-Polymorphism Not Implemented

Spec: §4.1 — "Generalize remaining free type variables into generic parameters (let-polymorphism)"

File: typeck.rs:70-73

Current: Documented as known limitation: "does NOT implement let-polymorphism... planned for v1.1"

Impact: let id = |x| x won't get a polymorphic type scheme. Users must manually annotate generic functions.

Recommendation: Acceptable for v1.0 since it's documented. Add to CHANGELOG and user-facing documentation. Priority for v1.1.

Severity: Major (spec gap) / Risk: Low (documented, planned)

⚠️ M-002: Trait Coherence / Full Resolution Not Complete

Spec: §4.3 — "At most one impl of a trait for a given type... Orphan rule: either the trait or the type must be local"

File: typeck.rs

Current: Simplified method lookup by name. No orphan rule enforcement. Documented as planned for v1.1.

Impact: Multiple conflicting trait impls won't be caught. Affects correctness when users define overlapping impls.

Recommendation: Add E2010 (ambiguous trait impl) error stub that fires if >1 impl found. Full coherence for v1.1.

Severity: Major (spec gap) / Risk: Medium (can cause silent incorrect resolution)

⚠️ M-003: DWARF Debug Info Deferred

Spec: §3.4 — "Emit DWARF debug info via LLVM DIBuilder: line tables, variable names, type descriptors"

File: docs/architecture/textual-ir-rationale.md:56-59

Current: Textual IR approach cannot emit !dbg metadata. Workaround: clang -g at IR level only.

Impact: Debuggers cannot show Axon source lines, only LLVM IR lines.

Recommendation: Acceptable per ADR trade-off. Milestone for v1.x: add textual !dbg metadata to .ll files. Document workaround in user guide.

Severity: Major (spec gap) / Risk: Low (correctness not affected, dev experience only)

Minor Findings (5)

ℹ️ m-001: Borrow Checker Uses AST-Walk, Not CFG Dataflow

Spec: §6.3 — "Use a simplified MIR-like control flow graph (CFG)"

Current: AST-walk analysis. Correctly catches common cases but may miss control-flow-dependent violations.

Documented: Yes, first 10 lines of borrow.rs explicitly state this and plan MIR-based migration.

Impact: Some complex borrow violations through branching may not be caught.

ℹ️ m-002: Runtime Shape Check Insertion Points Are Basic

Spec: §5.3 — "Insert runtime shape check at call site" for dynamic dims

Current: axon_tensor_shape_check declared and implemented as stub. Actual insertion at dynamic tensor op sites is minimal.

Impact: Dynamic tensor shape mismatches may not be caught at runtime.

ℹ️ m-003: User-Defined FFI Not End-to-End

Spec: §6.3 — "unsafe fn printf(fmt: &String, ...) → Int32; // C FFI declaration"

Current: CallingConv and ABI framework exist. Runtime C functions work. User-declared unsafe fn with no body not fully connected to LLVM extern declaration generation.
